# Traffic Route Viz Roadmap（2026H2 - 2027H1）

本路线图基于当前实现能力与已知问题，按“先稳定、再增强、后平台化”推进。

## 目标与指标

- 稳定性：P0/P1 缺陷平均修复时长 < 3 天。
- 可用性：复杂拓扑（>300 节点）可交互帧率保持可用，核心操作不阻塞。
- 可维护性：核心规则变更均有对应单测/e2e与文档沉淀。
- 协作效率：需求转 issue 标准化，Roadmap 与 GitHub Issue 双向同步。

## 阶段规划

### Phase 1（2026 Q3）：稳定性与可编辑性加固

- 完成自动边/手写边/节点编辑契约统一（选择、删除、重连）。
- 增加“边渲染健康检查”（handle 引用一致性）防回归。
- 优化跨分区连线可读性（左右方向、折线路由、减少绕线）。
- 导出链路稳定（全图导出、极端缩放场景、空图场景）。

交付定义：
- 所有编辑能力相关 e2e 通过；
- Spec 文档补齐约束 register；
- 关键缺陷回归为 0。

### Phase 2（2026 Q4）：语义增强与诊断能力

- Ingress/VS/Gateway 关联可解释化（显示“为何连线/为何未连线”）。
- 路径匹配诊断面板（Host/Path/Regex 相交调试）。
- 增加拓扑“最短路径/关键路径”高亮。
- 支持更多数据源（EndpointSlice、Gateway API 评估性支持）。

交付定义：
- 新增语义规则都有可视化解释与测试；
- 诊断功能可复现 80% 以上“连线疑问”场景。

### Phase 3（2027 Q1）：协作与平台化

- 图谱 Diff（基线与变更对比）。
- 团队共享模板（颜色主题、布局模板、导出模板）。
- 大图性能专项（虚拟化、按需渲染、批量操作优化）。
- CI 自动产出可追踪构建工件与回归报告。

交付定义：
- 大图性能有量化基线；
- Diff 与共享能力进入可用状态；
- 发布流程与规范闭环。

## 需求池（持续滚动）

- 交互：撤销/重做、批量选择、批量删除、锁定节点。
- 可视：连线拥塞分流、自动避障、分层导航地图。
- 导出：高分辨率分片导出、SVG 导出、PDF 导出。
- 规则：跨 namespace 策略开关、规则优先级可配置。
- 集成：Issue 自动回写、变更审计、团队通知。

## 依赖与风险

- 依赖：React Flow 版本能力、浏览器 CSS/缩放兼容、YAML 输入质量。
- 风险：规则持续演进导致复杂度膨胀；大图场景下交互退化。
- 缓解：约束 register + 测试矩阵 + 小步迭代 + 文档先行。
