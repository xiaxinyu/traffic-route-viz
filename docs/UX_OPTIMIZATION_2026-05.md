# UI/UX 深度优化沉淀（2026-05）

本文记录本次针对 `traffic-route-viz` 的界面与交互深度优化方案、实现位置、测试策略与回归点。

## 1. 目标与原则

- 提升操作效率：让“导入 → 解析 → 定位 → 排障 → 导出”形成闭环。
- 提升可读性：减少信息噪音，强调状态与关键动作。
- 提升专业感：统一视觉语言，减少试探性操作。
- 不破坏既有 Harness 约束：画布语义与资源关系规则保持不变。

## 2. 本轮落地清单

### 2.1 全局交互能力增强（`web/src/App.tsx`）

- 顶部新增“工作台状态条”：
  - 节点数、边数、自动边/手写边统计
  - 最近刷新时间与解析状态（正常/有告警）
- 搜索与定位：
  - 支持节点全文搜索（id/type/data 字段）
  - 提供“上一个/下一个”匹配跳转，自动居中到目标节点
- 类型筛选：
  - 支持按节点类型快速过滤（分区/入口/Host/Route/Service 等）
  - 每个筛选项显示当前数量
- 适配视图：
  - 提供“适配视图”按钮，快速恢复整体视角
- 选中详情：
  - 左侧面板新增“当前选中”区块，显示节点 id/type

### 2.2 可视化表达增强（`web/src/graphViewState.ts` + `web/src/App.tsx`）

- 搜索/筛选命中后：
  - 命中节点高亮（外发光 + 阴影）
  - 非命中节点降噪（降低不透明度、降低饱和）
  - 命中相关边高亮为强调色并提升线宽
  - 非命中边弱化

### 2.3 视觉系统升级（`web/src/index.css`）

- 引入 CSS 变量体系：
  - 语义化颜色变量（背景、文本、分割线、品牌色、警告色）
- 背景与层次：
  - 采用双径向渐变 + 线性底色，避免单调纯色背景
- 字体策略：
  - 使用更具产品感的跨平台字体栈（Avenir Next / Segoe UI / PingFang SC）
- 移动端适配：
  - 小屏下左右布局自动改为上下布局，侧栏限制高度并可滚动

### 2.4 工具条一致性优化（`web/src/DiagramActions.tsx`）

- 配色从单紫色调整为与主工作台一致的青绿色品牌语义
- 字体与边框细节统一
- 保留原有导出/保存/删除/打开功能与测试标识

## 5. 回归关注点

- 画布拖拽/手写连线/重解析保留手写边能力不可退化
- 大 YAML 文件导入时，搜索与筛选不应明显卡顿
- 移动端下左侧面板滚动与画布缩放不冲突
- 解析告警出现时，状态条与告警区域的信息一致

## 2026-05-08 补充：左侧面板减重优化

- 将“图谱筛选”从多按钮墙改为单一类型下拉 + 关键指标提示，降低认知负担。
- 删除“当前选中”独立卡片，避免与顶部状态及画布选中语义重复。
- 强化“图谱聚焦”动作闭环：`类型筛选 + 顶部搜索 + Enter 跳转`。
- 收窄左侧宽度并压缩间距，释放画布可视面积，提升主任务占比。

## 2026-05-09 补充：一级操作与编辑闭环

### 需求 / 缺陷拆解

- `[REQ-UX-01]` 画布高频动作一级可达：导出 PNG / Mermaid / draw.io、保存/打开画图文件、边标签、适配视图、删除选中连线/元素均直接暴露。
- `[REQ-UX-02]` YAML 编辑器提供就地操作：行数/文档数/字符数统计、解析刷新、清空、恢复示例、放大查看。
- `[REQ-UX-03]` 画布选中状态可见：顶部状态条和画布工具条同步显示已选节点/边数量。
- `[BUG-UX-01]` 删除工具隐藏过深：原右上折叠菜单导致删除连线/元素需要展开后定位，已改为一级入口。
- `[BUG-UX-02]` 空 YAML 可触发解析：空内容时就地解析按钮禁用，避免无效刷新。

### 测试补充

- 单元测试：`buildSelectionMetrics` 覆盖选中节点/边统计；`buildYamlTextStats` 覆盖多文档、空白文本统计。
- E2E：`smoke.spec.cjs` 覆盖一级画布工具条、YAML 就地清空/恢复；`edge-editing.spec.cjs` 覆盖选中计数与删除入口；`export-png.spec.cjs` 覆盖无需展开菜单即可导出 PNG。
